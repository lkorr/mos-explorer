# MOS / Temperament Finder ŌĆö Full Codebase Context

## What This App Is

A single-page browser app for exploring **MOS (Moment of Symmetry) scales** and **musical temperaments** across the generator spectrum. It targets microtonal music theorists and composers. The domain is xenharmonic (non-12-tone) music theory.

**Key concepts the LLM must understand:**
- **MOS scale**: A scale with exactly two step sizes (Large and Small), generated by repeatedly stacking a single interval (the "generator") and wrapping within a "period" (usually an octave = 1200 cents). Described as "NL Ms" (e.g. "5L 2s" = diatonic scale = 5 large + 2 small steps).
- **Generator**: The interval (in cents) stacked to produce the scale. Different generator sizes within a MOS region produce different L:s ratios (hardness).
- **Period**: Usually 1200 cents (one octave), but can be sub-octave (600, 400, 300, 240, 200, ~171.4 cents) or multi-octave (1800, 2400, 4800 cents).
- **Temperament**: A tuning system that tempers out specific commas (small JI intervals). Each temperament occupies a range of generator values within a MOS region.
- **EDO**: Equal Division of the Octave (e.g. 12edo = standard tuning). EDO generators appear as specific rational points within MOS regions.
- **Hardness (L:s ratio)**: How different the two step sizes are. 1:1 = equalized (EDO), approaching infinity = collapsed (merges with parent MOS).
- **Stern-Brocot tree**: The mathematical structure organizing all MOS patterns. Each MOS has a parent (fewer notes) and two children (more notes).

## Visual Layout (What the Display Looks Like)

The app fills the entire viewport with a dark theme (#1a1a2e background). From top to bottom:

### 1. Controls Bar (top)
A horizontal bar with controls:
- **Size N** (min/max): Two range sliders controlling which MOS sizes are shown (e.g. show only 2-note through 30-note scales)
- **Max EDO**: Range slider controlling the largest EDO whose generators are displayed

- **Reset Zoom**: Clickable label
- **Undo/Redo**: Arrow buttons (Ctrl+Z / Ctrl+Y)
- **Share Scale**: Generates a URL with current state (only visible when a generator is set)
- **? Help**: Opens help modal

### 2. Info Bar
Shows current view range, cursor position in cents, and basic instructions.

### 3. Main Canvas Area (fills remaining space)
This is where all the visualization happens, rendered on a `<canvas>` element:

#### a) Cent Axis (horizontal number line at ~y=44)
A horizontal bar showing generator size in cents. Tick marks and labels. This is the primary navigation element ŌĆö users click/drag on this to set the generator.

#### b) Temperament Bars (below axis, only when a MOS is selected)
Colored horizontal bars showing which temperaments span which generator ranges within the selected MOS. Each bar is labeled with the temperament name. Clicking opens the xen.wiki page.

#### c) MOS Region Bands (main area)
Colored horizontal bands, one row per MOS size (N). Each band spans the generator range where that MOS pattern exists. Left labels show the note count. Bands are colored by the Stern-Brocot tree structure:
- When nothing is selected: all bands visible at moderate opacity
- When a MOS is selected: that band is bright, others are dimmed
- Selected band color interpolates between two colors based on generator position (equalized vs collapsed end)

#### d) EDO Markers (above MOS bands, only when selected)
Vertical tick marks showing EDO generator positions within the selected MOS range. Labels show step\edo notation (e.g. "3\22"), cent value, and hardness ratio.

#### e) Generator Marker (vertical dashed line)
Red dashed vertical line showing the current generator position, with cent label at top.

#### f) MOS Circle Diagram (bottom-left overlay)
A circular "pie chart" visualization of the current scale:
- Each pie slice = one scale step, sized proportionally
- Large steps and small steps are colored differently
- JI ratio markers around the outside (gold tick marks + labels like "3/2", "5/4")
- Pitch dots on the circle perimeter (clickable to play notes)
- Glow animation on dots when notes are playing
- Title line above shows: MOS label, hardness ratio, UDP (up-down pattern), mode name
- Mode rotation buttons (gen ŌŚĆŌ¢Č and step ŌŚĆŌ¢Č) to cycle through modes
- Hardness slider between "equalized" and "collapsed"
- Parent Ō¢▓ / Child Ō¢╝ navigation arrows
- Step pattern display (e.g. "LLsLLLs")
- Center shows temperament name when applicable
- Resizable by dragging corner grip

### 4. Selected Panel (bottom-right, only when a MOS is selected)

#### Left Column: Synth Controls
- Waveform selector (sine/triangle/sawtooth/square)
- Volume slider

- Header showing current generator cents, hardness, and temperament name

- **Degree rulers**: One per scale degree, each showing the interval size in cents, nearest JI ratio, and a ruler visualization with JI markers
- Sort toggle: by scale degree or by generator chain order

## Module-by-Module Breakdown

### mos-data.js ŌĆö Static Data

**TEMPERAMENTS array** (~70 entries): Each entry has:
- `name`: Display name (e.g. "Meantone", "Mavila", "Porcupine")
- `mos`: String or array of MOS keys this temperament applies to (e.g. `['5L_2s', '7L_5s', '12L_7s']`)
- `genRange`: `[low, high]` cent range for the generator
- `commas` / `commaNames`: JI ratios tempered out and their names
- `wikiUrl`: Link to xen.wiki article
- `description`: HTML description (may contain `<br>`, `<a>`, `<i>` tags). These descriptions are opinionated and humorous.
- `period` (optional): Non-1200 period (e.g. 600 for half-octave temperaments like Pajara)
- `disabled` (optional): If true, temperament is hidden

**TEMP_COLORS**: Map of temperament name ŌåÆ hex color for display.

**MOS_NAMES**: Map of MOS key (e.g. "5L_2s") ŌåÆ family name (e.g. "Diatonic").

**MULTI_PERIOD_MOS_NAMES**: Names for multi-period MOS patterns (e.g. "2L_2s" ŌåÆ "Double Trivial").

**MODE_NAMES**: Map of MOS key ŌåÆ array of mode names ordered by UDP (brightest to darkest). Only defined for common MOS patterns (5L 2s has Lydian through Locrian, etc.).

## Key Implementation Details for Feature Planning

### State Management
All state is in the global `S` object. Changes flow: user action ŌåÆ update `S` ŌåÆ call `updatePanel()` + `render()`. Undo captures snapshots of key state fields. No reactivity system ŌĆö everything is imperative.

### Rendering Pipeline
Single `render()` call redraws everything. No partial updates. Canvas 2D with manual DPI scaling. Hit testing is done by storing drawn geometry (e.g. `S._drawnRegions`, `S._edoMarkers`, `S._circleDots`, `S._tempBars`) during render and checking against them on mouse events.

### Audio Architecture
Web Audio API. One oscillator per note. DynamicsCompressor ŌåÆ master GainNode ŌåÆ destination. ADSR is implemented manually with gain automation. Notes are tracked by keyboard code in `SYNTH._activeKeys`. The `updateHeldNotes()` function handles live retuning when the scale changes while notes are held ŌĆö it uses a greedy algorithm to reassign each held note to the nearest available degree.

### Data Flow for MOS Regions
`buildMOSRegions()` uses recursive Stern-Brocot tree traversal. Each call to `traverse(lp, lq, rp, rq, depth)` computes the mediant fraction and creates two MOS regions (one for each side of the mediant). Regions are automatically mirrored (for generator > period/2). The `perPeriodKey` vs `key` distinction handles sub-octave periods where the displayed key includes period repetitions.

### Temperament Matching
Temperaments are matched by:
1. Period must match
2. MOS key must be a direct match, descendant, or ancestor of a temperament's MOS list
3. Generator must be within the temperament's genRange (or its mirror)

### Scale Workshop Integration
Generates URLs for [Scale Workshop](https://scaleworkshop.plainsound.org/) with the current scale. Uses EDO step notation (e.g. "3\22") when the generator exactly matches an EDO, otherwise uses cent notation. Handles octave stretch and non-standard periods.

### Persistence
- Synth settings: `localStorage` key `mos-synth`
- URL state: Query params (period, mos, gen, mode, stretch)
- No other persistence

### Performance Considerations
- Chord finding uses binary search on pre-sorted JI pool for O(log n) interval lookup
- Ruler drawing is throttled by only rebuilding HTML structure when the MOS key changes
- Glow animation uses requestAnimationFrame loop that auto-stops when no glows are active
- MOS regions are rebuilt only when size/period controls change
- Canvas renders are not throttled (called freely on state changes)

## Conventions and Patterns

- All coordinates use "pure cents" internally; `stretchCents()` is applied only at display/audio time
- The `perPeriodSize` vs `size` distinction: `size` includes period repetitions (e.g. 14 notes for 7L 7s in half-octave), `perPeriodSize` is per-period (7)
- Color interpolation between `collapsedColor` and `equalizedColor` based on generator position
- Wiki links go to `en.xen.wiki` for temperaments, commas, intervals, and MOS patterns
- Descriptions in mos-data.js use inline HTML and contain personality/humor from the author
- The app uses the convention that "dark" generators (< period/2) have inverted UDP counting
